Présentation du Projet
Concept, stack technique et organisation du code
SESAME est une application de prescription rémunérée en points. Les Ambassadeurs (réceptionnistes, barmen, concierges) commandent des véhicules VTC pour leurs clients. En échange, ils accumulent des points SESAME échangeables contre des récompenses chez des fournisseurs partenaires.
📍
Concept Clé
  • Ambassadeur commande un VTC pour son client
  • Le chauffeur valide un code 4 chiffres = départ juridique
  • L'ambassadeur gagne des points (1pt / 10€)
  • Les points s'échangent contre des bons cadeaux QR code
  • Le fournisseur scanne le QR code pour valider la prestation
🌍
Lancement
  • Beta lancée à Montpellier
  • Cible initiale : hôtels, bars, restaurants
  • Modèle : frais 20% SESAME sur CA chauffeur
  • Hébergement prévu : Vercel + Railway
  • Applications iOS + Android via EAS Build

Rôles & Fonctions
Les 4 profils de l'écosystème SESAME
🎖️
Ambassadeur
📱 Application Mobile — iOS & Android
Réceptionniste, barman, concierge ou professionnel de l'accueil. Il commande des VTC pour ses clients et accumule des points SESAME à chaque course validée.
  • Tableau de bord avec anneau SVG de progression de niveau
  • Commander un VTC immédiat ou réserver à l'avance
  • Suivi en temps réel : code 4 chiffres affiché en permanence
  • Boutique de récompenses — échange de points contre bons
  • Bons cadeaux QR code avec alertes J-7 et J-1
  • Parrainage d'autres ambassadeurs (code unique 6 car.)
  • 4 niveaux : STARTER → PRO → ELITE → BLACK
  • Chat temps réel avec le chauffeur
  • Profil : thème nocturne/clair/auto + langue FR/EN/IT/ES
  • Sous-comptes Moral (Entreprise) — équipe + commissions
  • Historique complet des courses et des échanges
🚗
Chauffeur VTC
📱 Application Mobile — iOS & Android
Chauffeur VTC indépendant. Il active ou désactive sa disponibilité, reçoit des missions et valide les courses via le code 4 chiffres. Aucune sanction pour refus.
  • Toggle ON/OFF disponibilité (bloqué si course active)
  • Alerte course entrante : anneau countdown SVG 10 secondes
  • Refus de course sans aucune pénalité (règle juridique)
  • Saisie du code 4 chiffres (pivot juridique et financier)
  • Chat temps réel avec l'ambassadeur
  • Gestion client absent — bouton contacter SESAME uniquement
  • Revenus : CA encaissé par période (frais SESAME jamais affichés)
  • Profil KYC : 11 types de documents à télécharger
  • IBAN + SIRET pour facturation Stripe hebdomadaire
  • Note moyenne des courses effectuées
🛡️
Administrateur SESAME
🌐 Dashboard Web — Navigateur (React + Tailwind)
Équipe interne SESAME. Gestion complète de la plateforme : validation KYC, arbitrage des litiges, gestion des partenaires et déclenchement des virements.
  • KPIs temps réel : CA, courses, ambassadeurs, chauffeurs
  • Graphique d'activité sur 7 jours
  • Validation des bons cadeaux en attente (→ envoi QR)
  • Refus de bons avec remboursement automatique des points
  • Gestion des chauffeurs : KYC, statut, taux individuel
  • Gestion des ambassadeurs : niveaux, sanctions, profil
  • Arbitrage client absent : moduler points + indemnisation
  • Chat : supervision et intervention avec badge SESAME
  • Blacklist : blocage silencieux par critères biométriques
  • Gestion des fournisseurs : créer, modifier, régénérer codes
  • Déclenchement des virements chauffeurs
  • Paramètres système globaux
  • Exports CSV complets (courses, ambassadeurs, finances)
🏪
Fournisseur Partenaire
🌐 Page Web Standalone — sesame-pro.com/valider
Spa, restaurant, boutique ou prestataire partenaire. Il dispose d'une page web dédiée pour scanner et valider les bons cadeaux QR code des ambassadeurs.
  • Page web accessible sur n'importe quel appareil
  • Accès via lien QR code + code secret 4 chiffres
  • Triple vérification serveur : code, appartenance, offre active
  • Blocage automatique après 3 tentatives échouées
  • 6 états distincts : saisie, erreur, bloqué, validé, déjà utilisé, expiré
  • Confirmation verte + récapitulatif de la prestation
  • Affichage du contact SESAME en cas de blocage
  • Fond clair (#F2F2F7) — adapté à l'usage professionnel

Ce qui Reste à Faire
3 intégrations tierces + déploiement complet
🔥
Firebase Cloud Messaging (FCM) Mobile À faire
Envoyer les 11 notifications push listées dans les specs (iOS + Android). Le backend possède déjà le fichier pushNotifications.ts et le service notifications.ts côté mobile — il faut connecter les vraies credentials Firebase.
  • Créer le projet Firebase Console et télécharger google-services.json + GoogleService-Info.plist
  • Configurer Expo EAS Build avec les fichiers Firebase
  • Implémenter l'envoi serveur via firebase-admin SDK dans pushNotifications.ts
  • Notifications Ambassadeur : CHAUFFEUR_ACCEPTE, CHAUFFEUR_ANNULE, CODE_VALIDE, COURSE_TERMINEE, BON_VALIDE, BON_EXPIRE_J7, BON_EXPIRE_J1, SANCTION_POINTS, CHAUFFEUR_ATTEND
  • Notifications Chauffeur : NOUVELLE_COURSE, INDEMNISATION
  • Notification persistante code visible si app en arrière-plan
  • Badge rouge sur icône application si non lue
  • Cron job J-7 et J-1 pour alertes expiration bons
🗺️
Google Maps API Mobile À faire
Intégration cartographique pour la géolocalisation temps réel du chauffeur, le calcul d'ETA et le geofencing à 300m pour débloquer le bouton "Terminer la course".
  • Créer la clé API Google Maps Console (Maps SDK for Android + iOS)
  • Installer react-native-maps via Expo Config Plugin
  • Afficher position chauffeur en temps réel sur carte (Chauffeur Home)
  • Calcul ETA + affichage "~X min" dans le message client et la modal course
  • Geofencing 300m : activer le bouton "Terminer" uniquement à proximité de la destination
  • Mode dégradé : si Maps hors ligne → saisie manuelle adresse, prix non calculable
  • Affichage itinéraire dans l'écran course entrante (10s countdown)
💳
Stripe — Facturation Chauffeurs Backend À faire
Facturation hebdomadaire automatique des frais SESAME (20% du CA). Cycle lundi 01h00 → 13h00 → mardi 01h00 avec suspension et réactivation par webhook.
  • Configurer compte Stripe + clés API (test → production)
  • Créer customer Stripe à l'inscription chauffeur (stripe_customer_id en BDD)
  • Cron lundi 01h00 : générer factures des frais semaine précédente
  • Cron lundi 13h00 : relance paiement si en attente
  • Cron mardi 01h00 : suspension chauffeur si toujours impayé
  • Webhook Stripe : réactiver chauffeur instantanément après paiement
  • Frais JAMAIS affichés dans les interfaces chauffeur (seulement dans les factures Stripe)

Règles Métier & Charte Graphique
Règles non négociables et identité visuelle
⚖️ Règles Juridiques Absolues
Refus de course
Aucune sanction, pénalité ou conséquence pour un chauffeur qui refuse une course. Risque de requalification en contrat salarié.
Course partagée
Strictement INTERDITE. 1 course = 1 véhicule = 1 code = 1 client. Aucun covoiturage.
Mot "objectif"
BANNI de toute l'interface chauffeur. Ne jamais l'afficher.
Vouvoiement
Obligatoire dans TOUS les messages envoyés au client.
💰 Règles Financières
Pivot code 4 chiffres
Les points ne sont crédités QUE si code_valide_at IS NOT NULL. Avant validation : 0 obligation, 0 frais, 0 compensation.
Calcul des points
Math.floor(montant / 10) — SANS PLAFOND. Jamais de maximum.
Frais SESAME
JAMAIS affichés dans les interfaces chauffeur. Visibles uniquement dans les factures Stripe.
Sanctions différées
Si solde insuffisant : différer silencieusement. Aucune notification avant exécution.
